# Операционная база знаний по SEO/GEO для финтех-сайта ВЭД (buyusdtinvoice.ru)

Дата: 26 июня 2026. Язык: русский. Назначение: машиночитаемая база знаний под автоматизированный ИИ-скил. Приоритет поисковых систем: Яндекс → Google.ru → ИИ-ответы (Поиск с Алисой/Нейро, Google AI Overviews, ChatGPT/Perplexity). Тематика — YMYL (деньги/платежи).

> **Контекст сайта (по факту анализа главной).** buyusdtinvoice.ru — это, по состоянию на июнь 2026, по сути одностраничный лендинг бренда «Buy_USDTs» (Москва-Сити): оплата инвойсов и поставщиков в Китай через USDT/Alipay/WeChat/UnionPay, калькулятор, отзывы, ссылка на отдельный `blog.html`. Сайт сильно перегружен коммерческими мета-ключами (keywords: «оплата инвойсов в Китай», «пополнение Alipay», «купить USDT», «криптообменник Москва-Сити» и т.д.) и геопривязкой к Москве. **Вывод для скила:** информационный блог фактически ещё не развёрнут как кластерная структура — это «зелёное поле». Конкуренты в выдаче по нише — vedmaster.ru, wtcpay.ru, aifory pro, Global Finance VED, агрегаторы (VedHonest, Exnode), а также бизнес-медиа (vc.ru, DTF, sostav.ru, «Клеверенс»). Они уже занимают инфо-запросы развёрнутыми гайдами — это и есть зона борьбы.

## 1. TL;DR (главные выводы)

1. **YMYL — сквозное ограничение.** Тематика «оплата инвойсов / международные платежи / обход блокировок SWIFT» попадает в зону «Your Money or Your Life» (YMYL Financial Security). И Google (Quality Rater Guidelines, ред. 11 сентября 2025, 182 стр.), и Яндекс применяют повышенные требования к экспертности, авторству и достоверности. **Скил НЕ может автономно публиковать юридические и платёжно-правовые формулировки — обязателен человеческий фактчек.**
2. **Поведенческие факторы (ПФ) в Яндексе — главный драйвер, но накрутка мертва.** Официальная позиция (Михаил Сливинский, амбассадор интернет-площадок в Поиске Яндекса, интервью SEOnews, апрель 2024, вербатим): *«Мы стараемся не учитывать фродовые сигналы, какими бы изощрёнными ни были техники спамеров, чтобы накрутка просто не работала по умолчанию»*; позиция повторена на BDD 2025: *«Мы победили ссылки, и поведенческие факторы мы победим»*. Скил растит ПФ только белыми методами (структура, скорость, ответ-в-первом-абзаце).
3. **ИИ-ответы съедают трафик — выигрывает цитируемость, не позиция.** Ahrefs (обновление 4 февраля 2026, 300 000 ключей, данные GSC дек. 2023 → дек. 2025): *«Google's AI Overviews now result in a 58% lower average clickthrough rate for top-ranking pages, up from 34.5% just eight months ago»*; CTR позиции №1 по AIO-ключам упал с 0,073 до 0,016 (−78%). Для нового инфосайта стратегия = «answer-first контент, попадающий в цитирование», а не «20 синих ссылок».
4. **Семантика — через новый Вордстат + Wordstat API.** Старый интерфейс закрыт 11.08.2025; есть бесплатный Wordstat API (заявка, OAuth, лимит 1000 запросов/сутки, 10 rps). Скил автоматизирует сбор частотностей и динамики.
5. **Автоматизируемо:** метаданные, структура, перелинковка, Schema, контент-брифы, мониторинг позиций/индексации через API (Яндекс Вебмастер API, GSC API, Wordstat API). **Не автоматизируемо:** фактчек YMYL, юридические формулировки легальности платёжных схем, финальная редактура.
6. **Технический фундамент обязателен:** Core Web Vitals (LCP ≤ 2,5 c, INP ≤ 200 мс, CLS ≤ 0,1 на p75), серверный рендеринг (чтобы ИИ-краулеры видели контент), корректный robots.txt с управлением ИИ-ботами (GPTBot, OAI-SearchBot, PerplexityBot, Google-Extended, YandexAdditional).
7. **Турбо-страницы больше НЕ инструмент.** Яндекс прекратил поддержку технологии Турбо (анонс 07.02.2025 в блоге Вебмастера, отключение ~апрель 2025, замена на 301-редирект на оригинал); Турбо-страницы Директа отключаются 29 мая. Закладывать их в скил нельзя.
8. **Реалистичный горизонт:** для молодого инфосайта в конкурентной финтеме первые цитирования в Нейро — 3–8 недель при выполнении условий (топ-30 + Schema + авторство + свежесть); выход СЧ/НЧ-кластеров в топ — кварталы. KPI строятся на росте видимости и цитируемости, а не на разовых позициях.

## 3. Алгоритмы и фильтры — карточки

**YATI (офиц., с 2020).** Трансформерная модель смысла. Оценивает соответствие интента запроса и контента, заголовки, фрагменты. Предшественники: Палех (2016, заголовки), Королёв (2017, контент). → Соответствовать: писать под интент, LSI/синонимы, чёткие H2-вопросы.

**Поведенческие сигналы + антифрод (офиц. позиция).** Яндекс **не учитывает** фродовые сигналы (Сливинский, SEOnews, апрель 2024). Скрытых санкций «за качество» нет — всё внутри ранжирования по полезности. → Риск: накрутка ПФ нарушает правила Яндекса; при детекте обесценивает поведенческий профиль, восстановление по оценкам сообщества — многомесячное. **НЕ использовать.**

**Баден-Баден (офиц., анонс 23.03.2017).** Текстовый фильтр за переоптимизацию/переспам. Два вида: запросозависимый (просадка 7–30 позиций по фразе) и хостовый (≈20–30 позиций на весь сайт, уведомление в Вебмастере). → Не «напихивать» ключи, плотность по здравому смыслу, тексты для людей.

**Минусинск (офиц., 2015).** Антиссылочный фильтр за SEO-ссылки. Просадка ~20–30 позиций. → Не покупать ссылки массово.

**АГС, переспам, переоптимизация, аффилиат, кликджекинг, накрутка ПФ** — прочие санкции; хостовые приходят уведомлением в Вебмастер, запросозависимые — нет.

> ⚠️ **Про названия алгоритмов.** «Тайфун», «Оригами» тиражируются SEO-блогами как «алгоритмы Яндекса 2025», но **официальных анонсов Яндекса с этими названиями нет** — это гипотезы/ярлыки сообщества. В базе знаний помечать «неподтверждённое».

**Google core updates / Helpful Content / spam (офиц.).** QRG-2025 (январь + 11 сентября 2025) ввели определение генеративного ИИ, 3 категории спама (scaled content abuse, expired domain abuse, site reputation abuse); массовый ИИ-контент без ценности = Lowest. Декабрьский 2025 core update бил по генерик-контенту. → Соответствовать Helpful Content.

## 4. Регламент работы с семантикой (под автоматизацию)

**Источник частотностей:** новый Вордстат (старый закрыт 11.08.2025; данные с лагом 1–2 месяца) + Wordstat API.

**Шаги скила:**
1. **Затравочное ядро.** Базовые маски ниши: «оплата инвойса в Китай», «перевод денег поставщику», «платёжный агент ВЭД», «оплата в юанях», «агентский договор ВЭД», «закрывающие документы импорт», «валютный контроль 2026», «оплата через USDT поставщику», «платёж в Китай без SWIFT», «оплата авто из Китая».
2. **Расширение** через Wordstat API (`/v1/topRequests` — топ популярных и похожих) + сторонние базы (Bukvarix, Keys.so).
3. **Снятие частотностей:** базовая, фразовая («в кавычках»), точная («!»). Базовую тянет API; уточнённые — парсер с операторами.
4. **Динамика/сезонность:** `/v1/dynamics` (weekly/monthly). Помечать пики (предновогодние закупки импорта, ажиотаж при изменении валютного регулирования).
5. **Кластеризация по топу (hard/soft):** группировать по пересечению URL в топ-10 Яндекса (порог hard — 3–4 общих URL). Инструменты с API: Topvisor, Serpstat, Rush Analytics, Key Collector/Just-Magic.
6. **Приоритизация:** матрица «частотность × конкурентность × близость к конверсии». Для молодого домена — старт с НЧ/микроНЧ и «длинных хвостов» вопросного типа.
7. **Разметка интента:** информационный / коммерческий / гео. Инфо → блог; коммерческие → посадочные с мягким CTA.

**Распределение типов частотности для инфосайта в конкурентной финтеме:** ядро на СЧ/НЧ информационных запросах-вопросах; ВЧ («оплата в Китай») — pillar-страницы с горизонтом кварталов.

## 5. Чек-лист on-page оптимизации статьи (бинарный «сделано/нет»)

**База:**
1. [ ] H1 содержит основной запрос кластера, один на страницу.
2. [ ] Прямой ответ/определение (definition-box) в первых 100–300 символах.
3. [ ] TL;DR-блок после вступления.
4. [ ] H2 в форме вопросов пользователя (из кластера и «похожих запросов»).
5. [ ] Списки и ≥1 таблица (сравнение способов оплаты, сроков, комиссий).
6. [ ] Title ≤ ~60 симв., Description ≤ ~160 симв., оба с ключом и УТП.
7. [ ] ЧПУ-URL (латиница/транслит), без параметров.
8. [ ] Внутренние ссылки на pillar и 2–4 кластерные статьи.
9. [ ] Плотность ключей естественная (без переспама — риск Баден-Баден).
10. [ ] Дата публикации и «последнее обновление» видимы и в Schema (dateModified).
11. [ ] Изображения с alt, сжатые (WebP).
12. [ ] FAQ-блок (видимый Q&A) — 3–6 вопросов, ответы 80–150 слов.

**Под YMYL-доверие (требует человека):**
13. [ ] Блок автора: имя, фото, квалификация (опыт в ВЭД).
14. [ ] Ссылки на первоисточники-регуляторы (ФТС, ЦБ РФ, текст 115-ФЗ, валютное законодательство).
15. [ ] Конкретные цифры с источником и датой («по данным X на дату Y»).
16. [ ] Дисклеймер о необходимости консультации со специалистом по конкретной сделке.
17. [ ] Корректные юридические формулировки легальности схем (агентский договор, USDT) — **проверено юристом/экспертом**.
18. [ ] About/Контакты/реквизиты доступны с любой страницы.

**Под ИИ-цитирование (GEO/AEO):**
19. [ ] Самодостаточный фрагмент-ответ, вырезаемый без контекста.
20. [ ] Фактическая плотность: цифры, даты, конкретика вместо «воды» (Information Gain).
21. [ ] Уникальные данные/кейс, которых нет у конкурентов.
22. [ ] Schema: Article + Organization + BreadcrumbList (+ FAQPage как описание Q&A).
23. [ ] Контент доступен в HTML без JS-рендеринга (SSR).
24. [ ] Статья нацелена в топ-30 Яндекса (условие попадания в Поиск с Алисой).

## 6. Чек-лист технического SEO (регулярная проверка скилом)

1. [ ] robots.txt валиден, не блокирует CSS/JS, указывает Sitemap.
2. [ ] Управление ИИ-краулерами настроено осознанно (см. ниже).
3. [ ] XML-sitemap актуален, отправлен в Вебмастер и GSC.
4. [ ] Canonical на каждой странице, нет дублей title/description.
5. [ ] Core Web Vitals в «зелёной» зоне на p75 (LCP ≤2,5c / INP ≤200мс / CLS ≤0,1).
6. [ ] Мобильная версия корректна (mobile-first индексация).
7. [ ] HTTPS, нет смешанного контента.
8. [ ] Серверный рендеринг основного контента.
9. [ ] Нет «фатальных» ошибок в Яндекс Вебмастере (блокируют индексацию).
10. [ ] Структурированные данные валидны (Rich Results Test / валидатор Яндекса).
11. [ ] Внутренняя перелинковка pillar/cluster, нет «сирот».
12. [ ] Контроль индексации: % страниц в поиске (Вебмастер API / GSC API).

**robots.txt — рекомендованная политика (инфосайт, цель — макс. ИИ-видимость):**
- Разрешить retrieval/citation-ботов: `OAI-SearchBot`, `ChatGPT-User`, `PerplexityBot`, `Claude-SearchBot`, `Googlebot` (он же управляет AI Overviews).
- `YandexAdditional` / `YandexAdditionalBot` — **оставить разрешёнными**, чтобы контент попадал в ответы Поиска с Алисой/Нейро. По официальной справке Вебмастера, чтобы информация НЕ использовалась в ответах Алисы AI, нужно добавить `Disallow` именно для этих агентов; изменение учитывается за 2–14 дней. Эти агенты надо называть явно — общий `User-agent: *` они могут игнорировать.
- Training-боты (`GPTBot`, `Google-Extended`, `ClaudeBot`, `CCBot`) — решение стратегическое; для роста бренд-знания в моделях обычно оставляют разрешёнными (блокировка GPTBot/Google-Extended не влияет на ранжирование в обычном поиске Google).
- Всегда закрывать служебные пути (`/admin/`, формы, личный кабинет) для всех.
- ⚠️ robots.txt — рекомендация, не firewall; для жёсткого контроля — WAF.

## 7. Реестр API и инструментов (с эндпоинтами)

|---|---|---|---|---|

| **Google Search Console API** | clicks, impressions, CTR, position по query/page/country/device/date | Да, v1. `POST https://www.googleapis.com/webmasters/v3/sites/{siteUrl}/searchAnalytics/query`; URL Inspection API | OAuth 2.0 (scope `https://www.googleapis.com/auth/webmasters[.readonly]`); service-account плохо работает с GSC. Лимиты: Search Analytics ~1200 QPM; URL Inspection 2000/день; данные с лагом 2–3 дня; до 25 000 строк/запрос; токен живёт 1 час (refresh) | Позиции/CTR в Google, регрессии, контент-гэпы |
| **Google Indexing API** | Отправка URL на (пере)индексацию | Да | OAuth (service account) | Ускорение индексации новых статей |
| **Topvisor** | Позиции (Яндекс/Google, гео), кластеризация по выдаче | Да, бесплатный API, оплата за запрос (от ~0,045 ₽/проверка в топ-10 Яндекса) | Ключ API | Ежедневный съём позиций по кластерам |
| **Serpstat** | Частотность, позиции, конкуренты, отсутствующие ключи | Да. Методы: `SerpstatKeywordProcedure.getKeywordsInfo`, `SerpstatDomainProcedure.exportPositions` | Токен; лимиты = числу строк ответа, ≤1000 на запрос | Конкурентный анализ, контент-гэпы |
| **Keys.so** | Ключи домена, конкуренты, пересечения, «пустые зоны» | Да | Ключ | Карта запросов конкурентов, гэпы |
| **Bukvarix** | Большая база ключей | Да | Ключ | Быстрое расширение ядра (данные могут устаревать) |
| **Click.ru / PromoPult** | Сбор семантики, инструменты | Да | Ключ | Вспомогательный сбор |

## 8. Разделение «автоматизируем / отдаём человеку»

**Скил делает сам:**
- Сбор/обновление семантики, кластеризация, контент-брифы (структура, вопросы, LSI).
- Генерация и валидация мета-тегов, заголовков, ЧПУ.
- Schema-разметка (Article, Organization, BreadcrumbList, FAQPage).
- Предложения по внутренней перелинковке (pillar/cluster).
- Мониторинг позиций, кликов, показов, индексации, ИКС, CWV через API.
- Детект «просевших» статей, триггеры на обновление, черновые драфты.
- Отслеживание цитируемости в ИИ-ответах (проверка набора запросов в Нейро/AI Overviews).

**Только человек (ревью):**
- **Фактчек YMYL** — все цифры, ставки, сроки, статусы законности.
- **Юридические формулировки** легальности платёжных схем (агентские расчёты, USDT, обход SWIFT) — НЕ давать гарантий, проверять с юристом.
- Подтверждение квалификации автора, блок экспертизы.
- Финальная редактура (убрать канцелярит, «воду», ИИ-генеричность, кликбейт).
- Решение о публикации/обновлении значимого YMYL-контента.

## 9. Метрики и пороги

**Что мерить (по кластерам):**
- Позиции по кластерам (Яндекс + Google) — Topvisor/Вебмастер/GSC API.
- Органический трафик и динамика — Метрика/GSC.
- CTR сниппета — Вебмастер/GSC (растить через title/description).
- Поведенческие — Метрика (отказы, глубина, время, дочитываемость).
- Доля цитирований в ИИ-ответах (Нейро, AI Overviews) — ручная/полуавтоматическая проверка набора запросов.
- Конверсия в заявку/лид (мягкий CTA).
- ИКС (тренд).

**Пороговые сигналы смены тактики:**
- Статья «не зашла»: нет роста позиций/показов за ~8–12 недель после индексации → доработка (расширение, ответ-в-первом-абзаце, обновление данных) либо пересмотр интента.
- Падение по всему сайту + уведомление в Вебмастере → проверка на хостовый фильтр (Баден-Баден/Минусинск/ПФ).
- Падение без уведомления, синхронно с другими сайтами ниши → вероятен апдейт алгоритма, не ошибка; не паниковать, усиливать качество.
- Падение одной страницы → запросозависимый фильтр или потеря актуальности → точечная доработка.

**Циклы:** мониторинг позиций — ежедневно/еженедельно; контент-аудит и актуализация YMYL — не реже раза в квартал (свежие даты/цифры = плюс к ИИ-цитированию); техаудит — ежемесячно.

## 10. Предлагаемый цикл работы скила (черновик алгоритма)

```
ТРИГГЕР (еженедельно / по событию):
  - новый кластер из семантики ИЛИ просадка позиций/трафика ИЛИ устаревание контента (>90 дней)
↓
СБОР ДАННЫХ:
  - Wordstat API (частотность, динамика) + Topvisor/GSC/Вебмастер (позиции, CTR)
  - анализ топ-10 конкурентов (структура, длина, гэпы)
↓
ДЕЙСТВИЕ:
  - сгенерировать/обновить контент-бриф (структура, вопросы, факты-плейсхолдеры)
  - подготовить мета, Schema, перелинковку
  - [СТОП-ГЕЙТ] передать на человеческий фактчек + юридическое ревью (YMYL)
↓
ПУБЛИКАЦИЯ (после ревью):
  - обновить sitemap, пнуть Indexing API / переобход в Вебмастере
↓
ПРОВЕРКА (через 2–8 недель):
  - позиции, показы, CTR, попадание в топ-30, цитирование в Нейро/AIO
↓
ОТКАТ/КОРРЕКЦИЯ при просадке:
  - если фильтр (уведомление) → откатить переоптимизацию/ссылки, запрос на пересмотр
  - если апдейт → не трогать резко, усилить E-E-A-T и полноту
  - если «не зашло» → доработать ответ-блок, данные, интент
```

## 11. Caveats (ненадёжность данных и риски)

- **Веса ПФ Яндекс не раскрывает.** Всё про «важность CTR/last click/глубины» — оценки сообщества, не официальные коэффициенты. Помечено в таблицах как «оценка».
- **Названия «Тайфун»/«Оригами»** — не подтверждены официальными анонсами Яндекса; гипотезы сообщества.
- **Цифры по AI Overviews CTR** — из исследований агентств, методологии разнятся; направление (падение CTR) единодушно, величина — нет. Конкретно: Pew Research Center (анализ 68 879 запросов от 900 взрослых США, март 2025) — клик по органике 8% при наличии AI-сводки против 15% без неё, по ссылкам внутри сводки кликают лишь 1%; Seer Interactive (сент. 2025) — падение органического CTR на 61%, платного на 68%; корроборация Authoritas (−47,5%). По РУ-выдаче Яндекса публичных строгих исследований мало (оценки конференций ETarget и агентств).
- **AI Overviews максимально бьют по инфо-запросам — а это профиль сайта.** Ahrefs: 99,2% ключей, запускающих AI Overviews, — информационного интента; Pew — 60% запросов со словами «who/what/when/why» и 53% запросов из 10+ слов выдают AI-сводку. Поэтому ставка на цитируемость, а не на клик, обоснована.
- **Wordstat показывает «грязную» частотность** без операторов — завышена в 3–10 раз; для планирования нужны уточнённые частоты.
- **ИКС — не прямой фактор ранжирования** (официально); корреляция с топом есть, причинность не доказана.
- **Турбо «29 мая»:** официальный баннер Директа даёт день-месяц без явного года; организационный/Вебмастерский Турбо отключён ~апрель 2025 (анонс 07.02.2025). Не путать два отключения.
- **Риски автоматизации:** массовая ИИ-генерация без ценности = риск под Helpful Content (Google) и под «полезность» (Яндекс); юридические ошибки в YMYL-финтеме = репутационный и правовой риск. Стоп-гейт на человека обязателен.
- **Запрещённые методы НЕ применять:** накрутка ПФ, покупные ссылки, дорвеи, клоакинг, переспам — нарушают правила, дают временный эффект и многомесячные санкции.
- **Юридическая оговорка:** материалы базы знаний не являются юридической консультацией; легальность конкретных платёжных схем (USDT, агентские расчёты, обход SWIFT) требует проверки профильным юристом и может меняться с законодательством.
